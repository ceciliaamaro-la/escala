📋 TipoEscala
Permanência, Sobreaviso — extensível
PK id INT
COL nome VARCHAR(60)
COL descricao TEXT
🗂️ Escala
Cabeçalho da escala mensal por tipo
PK id INT
FK tipo_escala_id INT
COL mes TINYINT // 1-12
COL ano SMALLINT
ENUM status ENUM // RASCUNHO, PUBLICADA
COL criado_em TIMESTAMP
UNIQUE (tipo_escala_id, mes, ano)
📌 EscalaItem
Um registro por dia × militar × escala
PK id INT
FK escala_id INT
FK militar_id INT
FK calendario_dia_id INT
COL observacao VARCHAR(200)
UNIQUE (escala_id, calendario_dia_id)
// garante 1 militar por dia/escala
📎 TipoIndisponibilidade
Férias, Licença, Missão, Dispensa...
PK id INT
COL nome VARCHAR(60)
COL descricao TEXT
🚫 Indisponibilidade
Períodos em que o militar não pode ser escalado
PK id INT
FK militar_id INT
FK tipo_indisponibilidade_id INT
COL data_inicio DATE
COL data_fim DATE
COL observacao TEXT
📊 Quadrinho
Contagem de serviços — base para balanceamento
PK id INT
FK militar_id INT
FK tipo_escala_id INT
FK tipo_servico_id INT
COL ano SMALLINT
COL quantidade INT // total de serviços
UNIQUE (militar_id, tipo_escala_id, tipo_servico_id, ano)

⚑ Regras de Negócio Mapeadas

  • UNIQUE (escala_id, calendario_dia_id) em EscalaItem garante exatamente 1 militar por dia por tipo de escala
  • Antes de escalar, verificar se o militar possui Indisponibilidade ativa para aquela data
  • CalendarioDia.origem_tipo = AUTO para seg–sex (preto), sáb–dom (vermelho) e feriados nacionais fixos
  • Feriados móveis (Carnaval, Corpus Christi etc.) são inseridos manualmente com origem_tipo = MANUAL
  • Quadrinho é atualizado a cada EscalaItem inserido/removido, separado por tipo_escala + tipo_servico + ano
  • O algoritmo de geração usa Quadrinho para priorizar militares com menor contagem no tipo correspondente
  • Escala.status RASCUNHO permite edição; PUBLICADA bloqueia alterações sem permissão especial
  • Soft delete em Militar (campo ativo) preserva histórico de escalas e quadrinhos anteriores
