👉 SE JÁ TEM O DJANGO INSTALADO:

   1. Copie os arquivos para sua app:
      - models.py → escalas/models.py
      - admin.py → escalas/admin.py
      - forms.py → escalas/forms.py
      - signals.py → escalas/signals.py
      - views.py → escalas/views.py

   2. Modifique settings.py:
      AUTH_USER_MODEL = 'escalas.UsuarioCustomizado'

   3. Modifique apps.py:
      def ready(self):
          import escalas.signals

   4. Execute:
      python manage.py makemigrations escalas
      python manage.py migrate

   5. Crie superusuário:
      python manage.py createsuperuser

   6. Acesse:
      http://localhost:8000/admin/

FLUXO DE DADOS
═════════════════════════════════════════════════════════════════════════════════

OM
 ├─► Divisão ├─► Militar (Posto + Especialidade)
 ├─► TipoServico ├─► CalendarioDia (cada dia do ano)
 ├─► Escala (mês/ano/tipo)
 │   └─► EscalaItem (militar + dia)
 │       └─ Signal: Quadrinho += 1
 │
 ├─► Indisponibilidade (férias, licenças)
 │   └─ Valida alocações
 │
 └─► Quadrinho (contagem)
     └─ Usado para balanceamento automático


═════════════════════════════════════════════════════════════════════════════════
VALIDAÇÕES IMPLEMENTADAS
═════════════════════════════════════════════════════════════════════════════════

✓ CPF: 11 dígitos, não todos iguais
✓ Idade: mínimo 18 anos
✓ Matrícula: UNIQUE por OM
✓ Data: fim >= início
✓ Escala: UNIQUE (OM, tipo, mês, ano)
✓ EscalaItem: UNIQUE (escala, dia) = 1 militar/dia
✓ Quadrinho: UNIQUE (militar, tipo, serviço, ano)
✓ Divisão: deve pertencer à mesma OM
✓ Indisponibilidade: máx 365 dias
✓ Status escala: protege publicadas
